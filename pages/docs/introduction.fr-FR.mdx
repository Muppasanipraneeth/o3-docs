# Introduction

O3 est un framework frontend qui exploite les technologies frontend modernes pour permettre aux développeurs de créer collaborativement de meilleures expériences utilisateur pour les implémentations OpenMRS.

O3 fournit une large gamme d'outils, de composants, d'API et de modèles qui peuvent être exploités pour créer des applications frontend pour OpenMRS. Il est conçu pour être modulaire, permettant aux développeurs de choisir les composants dont ils ont besoin pour créer leurs applications frontend. O3 est également conçu pour être extensible, permettant aux développeurs d'étendre le framework avec leurs propres composants et API.

## Principes directeurs

O3 est guidé par les principes suivants:

- **Réutilisation du code** et **réduction des efforts dupliqués** - composants, API et modèles réutilisables.
- **Architecture modulaire** - permet l'ajout, la suppression ou la mise à jour facile des modules pour étendre les fonctionnalités ou adapter le système à des besoins spécifiques.
- **Configuration d'abord** - chaque module frontend peut spécifier ses propriétés de configuration.
- **Utilisabilité** - IU optimisée pour le point de soins, la réactivité et les dossiers de soins arbitrairement complexes.
- **IU améliorée** - IU plus intuitive, conviviale et attrayante visuellement.
- **Technologies modernes** - plus attrayant pour un pool de talents de développeurs mondial.
- **Interopérabilité améliorée** - prise en charge de FHIR et d'autres normes.

## Ce qu'O3 n'est pas

O3 n'est **pas** une nouvelle version de plateforme d'OpenMRS. O3 est un framework frontend que vous pouvez exécuter par-dessus votre base de données existante. Il communique avec la même API REST et l'API FHIR avec laquelle communique l'IU O2. Bien qu'O3 nécessite une version de plateforme minimale et des versions minimales de divers modules backend pour s'exécuter, vous n'avez pas besoin de refondre votre backend existant pour exécuter O3. Si vous êtes intéressé par l'exécution d'O3 par-dessus votre installation existante, suivez cette [recette](/docs/recipes/set-up-o3-instance) pour configurer une instance d'O3.

## Stack technologique

O3 utilise les technologies suivantes:

- [React](https://react.dev) - bibliothèque JavaScript pour la construction d'interfaces utilisateur
- [TypeScript](https://typescriptlang.org) - sur-ensemble fortement typé de JavaScript qui ajoute des annotations de type optionnelles à JavaScript, ce qui permet une vérification de type statique et un meilleur support d'outils.
- [Système de conception Carbon](https://carbondesignsystem.com) - un système de conception open source d'IBM.
- [Single SPA](https://single-spa.js.org) - routeur JavaScript pour les microservices frontend.
- [Webpack](https://webpack.js.org) - emballeur de modules pour applications JavaScript.
- API backend [OpenMRS REST](https://rest.openmrs.org) et [FHIR](https://om.rs/fhir).
