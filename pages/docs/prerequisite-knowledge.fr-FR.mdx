# Connaissances préalables [#prerequisite-knowledge]

Avant de commencer à construire avec O3, vous devez être familier avec au moins HTML, CSS et JavaScript. Si vous êtes totalement nouveau dans le développement frontend, il est particulièrement important que vous développiez une bonne compréhension des fondamentaux. Voici une liste des technologies que nous utilisons dans O3 et quelques ressources recommandées pour les apprendre.

## JavaScript [#javascript]

Connaître JavaScript est une condition préalable à tout le reste. Nous recommandons de passer par les ressources suivantes:

- [Introduction à JavaScript - Codecademy](https://www.codecademy.com/learn/introduction-to-javascript)
- [JavaScript à connaître pour React - Kent C. Dodds](https://kentcdodds.com/blog/javascript-to-know-for-react)
- [Le tutoriel JavaScript moderne](https://javascript.info/)
- [JavaScript pour les programmeurs impatients](https://exploringjs.com/impatient-js/toc.html)
- [Recherche d'opérateurs de Josh Comeau](https://www.joshwcomeau.com/operator-lookup/)
- [Guide visuel des références en JavaScript](https://daveceddia.com/javascript-references/)

## React [#react]

La [documentation officielle](https://react.dev/) est un excellent moyen de se familiariser avec React. **Ne survolez pas [les fondamentaux](https://react.dev/learn/thinking-in-react)**! Avoir une solide compréhension de ses concepts de base est très important pour vous aider à apprendre à développer des modules frontend de haute qualité dans O3. Nous recommandons également de passer par les ressources suivantes:

- [Erreurs courantes avec React - Josh Comeau](https://www.joshwcomeau.com/react/common-beginner-mistakes/)
- [Comprendre useMemo et useCallback](https://www.joshwcomeau.com/react/usememo-and-usecallback/)
- [Écrire des composants résilients - Dan Abramov (daté mais toujours très utile)](https://overreacted.io/writing-resilient-components/)
- [Un guide (presque) complet du comportement de rendu de React - Mark Erikson](https://blog.isquaredsoftware.com/2020/05/blogged-answers-a-mostly-complete-guide-to-react-rendering-behavior/)
- [Pourquoi React Re-renders - Josh Comeau](https://www.joshwcomeau.com/react/why-react-re-renders/)
- [Guide du débutant à React - Kent C. Dodds (daté mais toujours très utile)](https://egghead.io/courses/the-beginner-s-guide-to-react)
- [Un guide visuel du rendu de React](https://alexsidorenko.com/blog/react-render-cheat-sheet/)
- [Guide des re-rendus de React - Nadia Makarevich](https://www.developerway.com/posts/react-re-renders-guide)
- [Quand React rend-il votre composant - Zhenghao hao](https://www.zhenghao.io/posts/react-rerender)

## TypeScript

La plupart de notre code est écrit en TypeScript. Nous recommandons de passer par les ressources suivantes:

- [Essentiels de TypeScript](https://www.totaltypescript.com/books/total-typescript-essentials)
- [TypeScript en 5 minutes](https://www.typescriptlang.org/docs/handbook/typescript-in-5-minutes.html)
- [Total TypeScript](https://www.totaltypescript.com/tutorials)
- [Feuilles de triche React + TypeScript](https://github.com/typescript-cheatsheets/react)

## Outils JavaScript

- [yarn](https://yarnpkg.com/) - nous utilisons yarn comme gestionnaire de paquets. En général, si le dépôt que vous regardez a un fichier `yarn.lock` à sa racine, il utilise yarn. Nous exploitons les [espaces de travail](https://yarnpkg.com/features/workspaces) de yarn pour la versioning et la publication de paquets dans nos monorepos.
- [turbo](https://turbo.build/repo) - nous utilisons `turbo` pour exécuter des scripts pendant le développement local et nos processus d'intégration continue.
- [swc](https://swc.rs/) - nous utilisons SWC pour compiler notre code JavaScript et TypeScript dans un format que tous les principaux navigateurs supportent.
- [webpack](https://webpack.js.org) - nous utilisons webpack pour regrouper notre code pour la production. Nous utilisons [webpack 5](https://webpack.js.org/blog/2020-10-10-webpack-5-release/), qui est la dernière version de webpack au moment de l'écriture.

## Git

Il est impératif que vous gardiez votre code dans un système de contrôle de version. OpenMRS utilise [Git](https://git-scm.com/). Vous devriez connaître les bases de l'utilisation de Git et GitHub. D'excellentes ressources d'apprentissage incluent:

- Apprendre [Git Branching](https://learngitbranching.js.org/)
- [Git et GitHub pour les débutants](https://www.youtube.com/watch?v=RGOj5yH7evk)
- [Dangit, Git](https://dangitgit.com/en)
- [Guide OpenMRS pour utiliser Git](https://wiki.openmrs.org/display/docs/Using+Git)

## OpenMRS [#openmrs]

OpenMRS fournit diverses API par lesquelles le frontend peut demander des données. Nous vous recommandons de vous familiariser avec la documentation de ces API ainsi qu'avec le modèle de données OpenMRS. Trouvez les liens pertinents ci-dessous:

- [Modèle de données OpenMRS](https://devmanual.openmrs.org/technology/datamodel/)
- [API REST](https://rest.openmrs.org/)
- [Wiki FHIR](https://om.rs/fhir)
