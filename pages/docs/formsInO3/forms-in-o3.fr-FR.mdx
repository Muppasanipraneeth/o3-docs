# Formes dans O3

## Comment construire des formulaires avec l'interface graphique de O3 Form Builder

Vos BAs ou Devs peuvent utiliser l'interface utilisateur de Form Builder pour mettre en place et maintenir les formulaires O3.

- Voir le guide de l'utilisateur complet pour le Form Builder ici: https://ampath-forms.vercel.app/docs/quickstart
- Les conditions préalables et les dépendances sont couvertes ici: https://ampath-forms.vercel.app/docs/developer-guide/run-form-engine-in-openmrs3#prerequisites
- Plus d'informations et d'instructions d'exécution du code directement dans le README du [ESM Form Builder app repo here](https://github.com/openmrs/openmrs-esm-form-builder#openmrs-esm-form-builder-app)
  - Par exemple, voici une liste complète des types de champs communs utilisés dans les formulaires, avec des exemples. Le JSON de ces exemples peut même être copié-collé dans le champ d'édition JSON: ampath-forms.vercel.app/field-types-reference
- Note: Certains éléments de logique sont plus délicats, comme la logique de validation. C'est là que votre équipe peut vouloir que les développeurs interviennent pour écrire les règles JSON nécessaires. La documentation sur la façon de gérer les cas d'utilisation plus délicats est couverte ici, par exemple Validating Fields: ampath-forms.vercel.app/validating-fields
