# Vue d'ensemble

Il existe plusieurs approches que les implémenteurs peuvent utiliser pour configurer leurs instances O3. Ces approches sont les suivantes:

- Configuration des métadonnées
- Configuration des modules frontaux individuels
- Configuration au niveau de la distribution

## Configuration des métadonnées

Les configurations de métadonnées sont chargées via le [module d'initialisation](https://github.com/mekomsolutions/openmrs-module-initializer) dans O3. Le module d'initialisation, ou iniz en abrégé, est un module qui permet aux implémenteurs de charger des métadonnées dans leurs instances OpenMRS par le biais de fichiers JSON. Les différents fichiers de configuration livrés avec O3 sont situés dans le répertoire `distro/configuration` du dépôt [distro reference application config](https://github.com/openmrs/openmrs-distro-referenceapplication/tree/main/distro/configuration). Le module Initializer prend en charge le chargement d'une grande variété de types de données et de métadonnées, y compris, mais sans s'y limiter:

- Concepts
- Les emplacements
- Types d'attributs de personne
- Propriétés globales
- Types de rencontres
- Types de visites
- Programmes
- Flux de travail des programmes
- États du flux de travail du programme

## Configuration des modules frontaux individuels

La configuration par application est disponible dans O3 via le [système de configuration intégré](/docs/configuration-system). Chaque module frontal peut définir son propre schéma de configuration dans un fichier `config-schema.ts` dans son répertoire `src`. Les propriétés spécifiées dans le schéma de configuration permettent aux implémenteurs d'adapter le comportement du module à leurs besoins. Le système de configuration permet également aux implémenteurs de spécifier des valeurs par défaut pour les différentes propriétés de configuration. 

Les implémenteurs sont en mesure d'apporter des modifications aux configurations des modules frontaux par le biais du panneau d'outils intégré de l'implémenteur. Une fois que vous vous êtes connecté à O3, cliquez sur la flèche centrée en bas de l'écran pour faire apparaître les outils de l'implémenteur. Vous pouvez également cliquer sur l'icône en forme de roue dentée dans la barre de navigation. Une fois l'outil ouvert, vous pouvez rechercher les propriétés de configuration en parcourant la configuration et les modifier à la volée. Notez que toute modification apportée à la configuration à l'aide des outils de l'implémenteur sera perdue lorsque vous actualiserez la page. Pour apporter des modifications permanentes à la configuration, vous devrez valider ces modifications dans la configuration de votre distro. Les outils d'implémentation vous permettent de télécharger un fichier de configuration temporaire contenant vos changements en cliquant sur le bouton `Télécharger la configuration temporaire`.

Lorsque vous travaillez sur un module frontend, toutes les modifications que vous apportez au fichier `config-schema.ts` sont accessibles par programmation via le hook `useConfig`, comme le montre l'extrait de code ci-dessous:

```ts
import { useConfig } from "@openmrs/esm-framework" ;

export default function GreetUser({utilisateur}) {
  const config = useConfig() ;
  const greeting = config.greeting + ', ' + user.name
  return <span>{greeting}</span>;
}
```

## Configuration au niveau de la distribution

Les implémenteurs peuvent configurer leurs instances au niveau de la distribution en utilisant les approches suivantes:

- Spécifier les propriétés de configuration dans le fichier `distro.properties`.
- Spécifier une configuration JSON dans votre distro.

Pour en savoir plus sur la façon de configurer votre instance O3, lisez le guide [distro configuration](/docs/configuring-o3/configuring-your-o3-instance).
