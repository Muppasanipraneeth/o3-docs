# Customizing your import map

An import map is a JSON file that maps module names to URLs. In this case, these modules are what are known as [frontend modules](/docs/frontend-modules/overview) or microfrontends in O3. The URL is the URL of the module's `index.js` file. The URL can be a relative path or an absolute URL. In this case, the URLs are relative to the location of import map file on the server where the frontend is hosted.

It gets built out from the modules specified in the `frontendModules` key of the distro reference application's [spa-build config](https://github.com/openmrs/openmrs-distro-referenceapplication/blob/720408a57c98f51355496cb990fd19a0a8ebfce8/frontend/spa-build-config.json) file.

Implementers have the ability to pick and choose which frontend modules to include in their instance. This is done by customizing the import map. You can add or remove frontend module URLs from `spa-build config` file and then rebuild the distro reference application. The import map will be updated accordingly.
